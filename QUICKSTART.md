# 🚀 快速开始指南

## 一、首次启动

### 1. 启动 Python 服务（后台）

在项目根目录打开终端：

```bash
cd services
./start.sh
```

首次运行会自动安装依赖（Flask、Playwright 等），大约需要 1-2 分钟。

服务启动后会在 `http://localhost:5005` 运行，**保持终端不要关闭**。

### 2. 启动前端服务

打开新终端，在项目根目录运行：

```bash
yarn dev
```

访问 http://localhost:3000

### 3. 配置孕期信息

1. 点击侧边栏的「设置」
2. 填写**末次月经日期**（推荐）或**预产期**
3. 点击「保存设置」

系统会自动计算当前孕周。

### 4. 配置小红书 Cookie（可选）

> 如果不配置 Cookie，可以使用「手动输入」模式

**获取 Cookie：**

1. 访问 https://www.xiaohongshu.com 并登录
2. 按 `F12` 打开开发者工具
3. 点击 `Application` → `Cookies` → `xiaohongshu.com`
4. 复制三个值：
   - `a1`
   - `web_session`
   - `webId`
5. 组合格式：`a1=xxx; web_session=xxx; webId=xxx`
6. 粘贴到设置页面的「小红书 Cookie 配置」框中
7. 点击「保存设置」

## 二、添加第一条内容

### 方式一：链接导入（需要 Cookie）

1. 打开小红书 App 或网页版
2. 找到一篇有用的孕期笔记
3. 点击分享，复制链接
4. 回到网站，点击**右下角 ➕ 按钮**
5. 选择「🔗 链接导入」
6. 粘贴链接，点击「开始导入」
7. 等待 3-5 秒，AI 自动分析完成

### 方式二：手动输入（无需 Cookie）

1. 打开小红书笔记
2. 复制标题和正文内容
3. 点击**右下角 ➕ 按钮**
4. 选择「✍️ 手动输入」
5. 粘贴标题和内容
6. 点击「开始导入」
7. AI 自动分析

## 三、查看内容

### 首页 - 当前孕周的精选内容

首页会自动展示：
- 当前孕周进度（如：孕19+3）
- 本周重点提醒
- 按类别整合的内容（营养、产检、运动等）
- 显示"来自 X 篇笔记"

**特点：**
- ✅ 自动去重
- ✅ 智能合并相似内容
- ✅ 只显示当前孕周相关的

### 我的内容 - 浏览所有笔记

点击侧边栏「我的内容」：
- 🔍 搜索：标题、内容、作者、标签
- 🏷️ 筛选：按类别（营养、产检等）
- 📅 筛选：按孕周
- 📊 排序：按时间或按孕周

**功能：**
- 查看原始笔记
- 查看 AI 提取的关键点
- 收藏、删除、查看原文

### 购物清单

AI 自动从笔记中提取推荐物品，按孕期阶段分类。

## 四、日常使用

**典型使用场景：**

```
刷小红书 → 看到有用内容 → 点击 ➕ → 粘贴链接 → 导入
                                    ↓
                            AI 自动分析整理
                                    ↓
                        首页自动显示相关内容
```

**使用技巧：**

1. **积少成多**：每天刷小红书时添加 2-3 条，一周后就有丰富的知识库
2. **当前优先**：首页默认展示当前孕周，最相关的内容优先看
3. **搜索查找**：想查某个话题，去「我的内容」搜索
4. **定期整理**：删除不需要的笔记，保持知识库高质量

## 五、故障排查

### Python 服务未启动

**症状：** 点击导入时提示"Python API 服务未启动"

**解决：**
```bash
cd services
./start.sh
```

### Cookie 失效

**症状：** 提示"Cookie 失效"或提取失败

**解决：**
1. 重新登录小红书
2. 获取新的 Cookie
3. 在设置页面更新

### 提取内容为空

**症状：** 提取成功但内容为空

**解决：**
1. 检查链接是否正确
2. 尝试在浏览器打开链接确认笔记存在
3. 使用「手动输入」模式

### 端口被占用

**症状：** Python 服务启动失败，提示 5005 端口被占用

**解决：**
```bash
# 查找占用端口的进程
lsof -ti:5005 | xargs kill -9

# 重新启动
cd services
./start.sh
```
